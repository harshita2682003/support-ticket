How to Create a new Database

Learn how to create and configure a new database for your application with our step-by-step guide

Creating a database is an essential step when setting up your application. This guide will walk you through the process of creating a new database in our system, configuring its settings, and getting it ready for use in your applications.

Note

Before proceeding, make sure you have the necessary permissions to create a database. If you don't have the right permissions, contact your system administrator.

Prerequisites

  • An active account with administrator privileges
  • Access to the admin dashboard
  • Knowledge of your desired database settings
  • Understanding of basic database concepts
1

Log in to the Dashboard

Start by logging into your admin dashboard with your username and password. This will give you access to all the administrative functions, including database management.

Tip: Make sure you're using a secure connection (https) when accessing your admin dashboard.

Fig. 1: Admin dashboard login screen
2

Access Databases Section

Once logged in, navigate to the DatabasesThis section contains all database management functions tab at the top of the dashboard.

This section contains all tools needed for creating and managing databases within the system. The database section is typically represented with a database icon.

Fig. 2: Locating the Databases tab in the dashboard
3

Create Database

Within the Databases section, look for the "Create Database" button or the "+ New Database" option. Click on this to begin the database creation process.

Note: You may need specific permissions to see the Create Database button. If you don't see it, contact your administrator.

Alternatively, you can create a database using SQL directly if you have access to the SQL console:

SQL Command 
CREATE DATABASE your_database_name;

-- If you need to specify character set and collation:
CREATE DATABASE your_database_name
CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;
4

Configure the Database

In the "Create database" section, you'll need to provide some information to set up your database:

  • Database Name: Choose a descriptive name for your database. Avoid spaces and special characters.
  • Character Set: The default is usually UTF-8, which supports multiple languages.
  • Collation: This determines how the data is sorted. The default is usually fine for most applications.
  • User Access: Configure which users or roles have access to this database.
Fig. 3: Database configuration form

Important

Choose a secure password for your database that includes a mix of letters, numbers, and special characters. Do not use easily guessable passwords.

5

Verify and Use

After creating your database, you should verify that it was successfully created and is accessible.

  1. Check that the database appears in your database list
  2. Try connecting to it using your preferred database client
  3. Verify that you can create tables and perform basic operations

Success

Your database is now ready to use! You can start creating tables and implementing your database schema.

Here's an example of a connection string you might use to connect to your new database:

Connection String Example 
"Server=myserver;Database=your_database_name;User Id=your_username;Password=your_password;"

Troubleshooting

If you encounter any issues while creating your database, check these common problems and solutions:

Permission Denied Error

Problem: You receive an error saying you don't have permission to create a database.

Solution: Contact your system administrator to grant you the necessary permissions. You need CREATE DATABASE privileges to perform this action.

Database Name Already Exists

Problem: You get an error saying the database name is already in use.

Solution: Choose a different name for your database. Database names must be unique within the server.

Connection Issues

Problem: You've created the database but can't connect to it.

Solution: Verify that the database user has been granted access to the new database. Check your connection string parameters, especially the server name, username, and password.
